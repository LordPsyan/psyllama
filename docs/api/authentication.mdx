---
title: Authentication
---

No authentication is required when accessing Psyllama's API locally via `http://localhost:11434`.

Authentication is required for the following:

* Running cloud models via psyllama.com
* Publishing models
* Downloading private models

Psyllama supports two authentication methods:

* **Signing in**: sign in from your local installation, and Psyllama will automatically take care of authenticating requests to psyllama.com when running commands
* **API keys**: API keys for programmatic access to psyllama.com's API

## Signing in

To sign in to psyllama.com from your local installation of Psyllama, run:

```
psyllama signin
```

Once signed in, Psyllama will automatically authenticate commands as required:

```
psyllama run gpt-oss:120b-cloud
```

Similarly, when accessing a local API endpoint that requires cloud access, Psyllama will automatically authenticate the request:

```shell
curl http://localhost:11434/api/generate -d '{
  "model": "gpt-oss:120b-cloud",
  "prompt": "Why is the sky blue?"
}'
```

## API keys

For direct access to psyllama.com's API served at `https://psyllama.com/api`, authentication via API keys is required.

First, create an [API key](https://psyllama.com/settings/keys), then set the `PSYLLAMA_API_KEY` environment variable:

```shell
export PSYLLAMA_API_KEY=your_api_key
```

Then use the API key in the Authorization header:

```shell
curl https://psyllama.com/api/generate \
  -H "Authorization: Bearer $PSYLLAMA_API_KEY" \
  -d '{
    "model": "gpt-oss:120b",
    "prompt": "Why is the sky blue?",
    "stream": false
  }'
```

API keys don't currently expire, however you can revoke them at any time in your [API keys settings](https://psyllama.com/settings/keys).
