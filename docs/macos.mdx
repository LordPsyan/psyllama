## System Requirements

* MacOS Sonoma (v14) or newer
* Apple M series (CPU and GPU support) or x86 (CPU only)


## Filesystem Requirements

The preferred method of installation is to mount the `psyllama.dmg` and drag-and-drop the Psyllama application to the system-wide `Applications` folder.  Upon startup, the Psyllama app will verify the `psyllama` CLI is present in your PATH, and if not detected, will prompt for permission to create a link in `/usr/local/bin`

Once you've installed Psyllama, you'll need additional space for storing the Large Language models, which can be tens to hundreds of GB in size.  If your home directory doesn't have enough space, you can change where the binaries are installed, and where the models are stored.

### Changing Install Location

To install the Psyllama application somewhere other than `Applications`, place the Psyllama application in the desired location, and ensure the CLI `Psyllama.app/Contents/Resources/psyllama` or a sym-link to the CLI can be found in your path.  Upon first start decline the "Move to Applications?" request.


## Troubleshooting

Psyllama on MacOS stores files in a few different locations.
- `~/.psyllama` contains models and configuration
- `~/.psyllama/logs` contains logs
    - *app.log* contains most recent logs from the GUI application
    - *server.log* contains the most recent server logs
- `<install location>/Psyllama.app/Contents/Resources/psyllama` the CLI binary

## Uninstall

To fully remove Psyllama from your system, remove the following files and folders:

```
sudo rm -rf /Applications/Psyllama.app
sudo rm /usr/local/bin/psyllama
rm -rf "~/Library/Application Support/Psyllama"
rm -rf "~/Library/Saved Application State/com.electron.psyllama.savedState"
rm -rf ~/Library/Caches/com.electron.psyllama/
rm -rf ~/Library/Caches/psyllama
rm -rf ~/Library/WebKit/com.electron.psyllama
rm -rf ~/.psyllama
```
